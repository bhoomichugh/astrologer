# AI Usage Declaration

This document declares how Artificial Intelligence (AI) was utilized in both the implementation of features inside the **Astrologer CRM** application and during the development/coding process.

---

## 🤖 1. AI Features within the Application

The Astrologer CRM incorporates native AI-assisted workflows to save time for practicing astrologers and enhance client notes.

### Summary Generation
- **Model**: `gemini-1.5-flash` (via the official Google Generative AI SDK, `@google/generative-ai`)
- **Location of Integration**: [aiController.js](file:///c:/Users/DELL/OneDrive/Documents/Astrologer/backend/controllers/aiController.js)
- **Workflow**:
  1. The astrologer conducts a consultation session and saves unstructured notes (covering planetary charts, client questions, or general discussion details).
  2. The astrologer clicks the "AI" button on the **Consultations** management panel.
  3. The server posts the notes to the `/api/ai/generate-summary` endpoint.
  4. The backend constructs a structured prompt instructing the model to summarize the notes professionally, isolate the client's primary concerns, and outline actionable recommendations.
  5. The generated text response is returned to the frontend and stored in the MongoDB database under the `aiSummary` field of the corresponding `Consultation` document.

## 🛠️ 2. AI Usage in Codebase Development

The development, design, refactoring, and documentation of this codebase were assisted by AI coding assistants (Gemini / Antigravity). 

### Scope of AI Coding Assistance
1. **Database Schema Design**: AI assisted in designing Mongoose database models (`User`, `Client`, `Consultation`, `Booking`, `ChatMessage`) to establish relationships, default values, and data validations.
2. **Full-Stack REST Architecture**: Scaffolded controllers, routes, and authentication middleware (such as password hashing via bcrypt and token verification via JWT) on the Express server.
3. **Frontend Portal Separation**: Generated separate portal modes in `main.jsx` and `App.jsx`, organizing the layout, state variables, and visual design assets.
4. **Theme Customization & Styling**: Developed variables and custom layout rules in `styles.css` to accommodate distinct visual themes (Orange branding for the Astrologer CRM, and Purple branding for the Client Portal).
5. **Documentation and Walkthroughs**: Generated technical project notes, setup steps, and markdown deliverables (`PROJECT_NOTES.md` and `AI_USAGE.md`).

### Developer/AI Collaboration
- Code generated by AI was reviewed, customized, and tested locally by human developers to guarantee safety, syntax correctness, and alignment with project goals.
- AI was used as a pair-programming partner to optimize API query performance, establish proper CORS authorization filters, and implement robust error boundaries in the frontend.
